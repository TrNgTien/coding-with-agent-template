# Context — agent rules

**Put:** stack, languages/versions, repo URLs, branch conventions, review/test rules, env vars (names only—no secrets), architecture one-pagers, eng team contacts, durable “how we work” prefs.

**Don’t put:** secrets, large paste dumps, transient tickets (use `Tasks/` or `Daily/`).

**Read when:** coding standards, repo layout, “what do we use for X?”

**Link:** `[[Projects/...]]` for deep dive per initiative.
