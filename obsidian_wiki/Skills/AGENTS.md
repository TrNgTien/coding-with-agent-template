# Skills — agent rules

**Put:** short refs for automations/skills: what to load, in what order. Link `[[Context/...]]` for standards; never fork duplicate policy text.

**Don’t put:** canonical standards (single source = `Context/`).

**Read when:** running or defining a repeatable agent workflow.

**Cursor host skills:** `~/.cursor/skills/` — see table on [[Skills/Skills hub]] (e.g. go-expert, js-expert).

**Vault edits / logs:** Changing notes under `Skills/` → root [[AGENTS]] **Obsidian logging (mandatory)** + **Vault edits (agents)**; personal skill **obsidian-vault-log** ([[Skills/Skills hub]]).

**Security:** No secret values in `Skills/` notes; see root [[AGENTS]] **Security (mandatory)** and personal **security-expert** skill ([[Skills/Skills hub]]).

**Scope:** Agents default to **back-end engineering** only; see root [[AGENTS]] **Agent scope (mandatory) — backend vs SRE** (no free SRE actions).

**Stale vault docs:** Users may change repos without agents — [[Skills/Skills hub]] **vault-doc-sync** (ask, then git sync / worktree).
